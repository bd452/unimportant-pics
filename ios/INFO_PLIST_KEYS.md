# Required iOS Info.plist additions

The full iOS project is generated by `npx react-native init` or
`react-native upgrade`. Once it exists, add the following keys to
`ios/unimportantpics/Info.plist` so the app can read and modify the user's
camera roll. Without these strings the OS will reject the permission request.

```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>Unimportant Pics needs read access to your camera roll so it can review photos and recommend which ones to delete.</string>

<key>NSPhotoLibraryAddUsageDescription</key>
<string>Unimportant Pics requests this permission so it can delete photos from your camera roll after you confirm.</string>
```

## Capabilities and pods

`@react-native-camera-roll/camera-roll`, `react-native-permissions`,
`react-native-gesture-handler`, and `react-native-reanimated` require native
linking. After installing JS packages, run:

```bash
yarn pods
```

`react-native-permissions` requires you to declare which permission handlers
ship in the binary. In `ios/Podfile` add:

```ruby
setup_permissions(['PhotoLibrary', 'PhotoLibraryAddOnly'])
```

(See the upstream README for the exact macro location.)
